# Crate Structure

Umbra is organized as a small set of Rust crates with clear
responsibilities. The kernel and architecture crates are
hardware-independent; each MCU adds its own `drivers` and `boot` crate
implementing a shared trait.

```
kernel (no_std, no external deps)
  |   +-- src/platform/mod.rs: PlatformBoot trait (PAL)
  |
  +-- arm (cortex-m, kernel)
  |     Architecture-agnostic ARMv8-M / ARMv8.1-M primitives:
  |     SAU, MPU, mmio (volatile R/W + barriers), vector tables
  |
  +-- drivers/<mcu>  (peripheral_regs, kernel)
  |     Platform-specific: RCC, GPIO, UART, DMA, HASH, AES, GTZC/RISAF,
  |     OTFDEC/MCE, OCTOSPI/XSPI
  |
  +-- boot/<mcu>  (arm, drivers/<mcu>, kernel, peripheral_regs, cortex-m)
        Binary crate. Entry point: secure_boot(). Implements PlatformBoot
        for the MCU. Handlers, API implementations, validator.
```

## kernel

Architecture-agnostic core logic:
- Enclave descriptor management
- Key storage server (key generation, derivation)
- Memory protection traits (`MemorySecurityGuardTrait`)
- Enclave Swap Space (ESS) data structures
- NSC API symbol declarations
- **`PlatformBoot` trait** in `src/platform/mod.rs` — the boot-time
  contract that each MCU implements (`init_clocks`, `init_gpio`,
  `init_uart`, `init_security`, `init_kernel`, `init_external_flash`,
  `configure_ns_boot`, `jump_to_ns`).

Optimized for size (`opt-level = "z"`).

## arm

ARM Cortex-M33 / Cortex-M55 hardware abstraction:
- **SAU driver** — Secure Attribution Unit region configuration
- **MPU driver** — Memory Protection Unit with ARMv8-M RBAR/RLAR format
- **mmio.rs** — volatile register read/write helpers with DSB/ISB barriers
  (used by every platform's peripheral driver to make ordering explicit)
- **startup.s** — vector table, exception handlers, `save_enclave_context`,
  SVC dispatch (M33 build; M55 reuses the same code path with optional
  Q-register save in lazy mode if MVE is used)

## drivers — per platform

### `stm32l552`
STM32L552/L562 peripheral drivers:
- **RCC** — clock gating for all peripherals
- **GPIO** — pin mode, alternate function, set/reset
- **UART** — LPUART1 (L552) / USART1 (L562) at 9600 baud
- **DMA** — 16-channel queue-based transfer manager
- **HASH** — SHA-256, HMAC with context save/restore
- **AES** — hardware (L562) or software emulated (L552)
- **GTZC** — MPCBB block-level SRAM security
- **OTFDEC** — on-the-fly decryption (L562 only)
- **OCTOSPI** — memory-mapped external flash (L562 only)

### `stm32n657`
STM32N657 peripheral drivers:
- **RCC** — IC1..IC20 kernel clock dividers, PLL1/2/3 setup, peripheral gating
- **GPIO** — port-based, NS-aliased base + AF up to 15
- **UART** — USART1 at 115200 baud (HSI 64 MHz, BRR=556)
- **HASH** — hardware HMAC-SHA256 (used for boot-time chained measurement
  over enclave + NPU bytecode/weights; replaces the SW implementation
  used during early bring-up)
- **AES** — `CRYP1` hardware engine
- **RISAF** — per-memory firewall (RISAF2 = AXISRAM1, RISAF12 = XSPI2, ...)
- **MCE** — memory cipher engine for external flash (currently passthrough;
  encrypt-at-rest "Path B-full" deferred — see `oracle.rs` doc)
- **XSPI / XSPIM** — memory-mapped XSPI2 read path (memory-mapped at
  `0x70000000`; programming via STM32CubeProgrammer external loader)

## boot — per platform

The binary crate for each MCU:
- `main.rs` — `secure_boot()` initialization sequence calling the
  `PlatformBoot` methods in order.
- `platform_impl.rs` — implementation of `PlatformBoot` for this MCU.
  This is where almost all per-board hardware setup lives.
- `secure_kernel.rs` — `Kernel` struct, ESS miss handling, block loading.
- `handlers.rs` — exception handlers (HardFault, MemManage, UsageFault,
  SecureFault, BusFault). On N657 it also surfaces IAC violations.
- `api_impl.rs` — NSC API implementations (`_imp` functions).
- `validator.rs` — HMAC verification + AES decryption (formal model analog).
- `raw_print.rs` — low-level UART output for exception contexts.
- (`stm32n657` only) `boot_measurements.rs` — generated by
  `tools/measure_blobs.py` to hold expected HMACs of NPU bytecode + weights.

## host/common

Shared C helpers that landed during the N657 port:

- `umbra_hex.{c,h}` — `umbra_u32_to_hex()` formatter used by all NS hosts.
- `umbra_mem.c` — minimal `memset` / `memcpy` for `-nostdlib` builds.

Both are included from `host/bare_metal_arm`, `host/freertos_arm`,
`host/bare_metal_n657`, `host/freertos_n657`, and
`host/object_detection_n657`.
