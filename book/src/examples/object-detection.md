# NPU Object Detection (STM32N657)

The `host/object_detection_n657/` example runs a quantized Tiny YOLO v2
person detector on the STM32N6 NPU **inside the Umbra Secure enclave**.
The model bytecode and weights are flashed into XSPI2 alongside the
host bin; the FSBL HMACs them at boot together with the enclave code
(chained measurement), and only valid blobs are made readable to the
enclave at runtime.

This is the most complete demonstration of Umbra: a real ML workload
that needs hardware (the NPU), driven by an enclave inside the Secure
World, with the host limited to passing an image in and reading
detections out via shared NS-visible buffers.

## Pipeline

```
NS host (FreeRTOS task)              Secure enclave (NPU runner)
-----------------------              -----------------------
load test image (RGB888 224Ă—224)
write image into INPUT_SHARED -----> read INPUT_SHARED
                                     kick NPU (EPOCHCTRL)
                                     poll EPC.IRQ
                                     write logits â†’ OUTPUT_SHARED
read OUTPUT_SHARED <----------------- enclave returns
host-side YOLO decode
print detections over UART
```

The shared regions live in AXISRAM1 NS view:

| Region | Address | Size | Direction |
|---|---|---|---|
| INPUT_SHARED | `0x24080000` | 4Ă—4 KB header + image | Host â†’ Enclave |
| OUTPUT_SHARED | `0x240C0000` | 4Ă—4 KB header + logits | Enclave â†’ Host |

Both are also visible from the Secure side via the AXISRAM1 Secure
alias (`0x34080000` / `0x340C0000`), with MPU regions 6 and 7 opened by
the FSBL so the enclave can read/write them without faulting.

## Build pipeline

`./rebuild_all.sh` with `HOST_APP=object_detection` runs the standard
build plus the NPU-specific pre-steps:

1. **Extract the model bytecode** from the ST Edge AI build product.
   `tools/extract_bytecode.py` reads
   `host/object_detection_n657/Model/NUCLEO-N657X0-Q/network_ecblobs.h`
   (the `_ec_blob_*[]` arrays produced by `stedgeai`) and concatenates
   them into `build/model_bytecode.bin`.
2. **Compute boot-time chained HMACs** over the bytecode and over
   `network_data.xSPI2.bin` (model weights). `tools/measure_blobs.py`
   writes the result into
   `src/hardware/platform/stm32n657/boot/src/boot_measurements.rs`,
   which the FSBL compiles in as expected reference values.
3. **Build the FSBL** (now containing `MODEL_BYTECODE_HMAC` and
   `MODEL_WEIGHTS_HMAC`). At boot, the FSBL reads bytecode and
   weights from XSPI2 via HW HMAC-SHA256 and aborts if either HMAC
   diverges (`[UMBRASecureBoot] HMAC FAIL` on UART). A typical full
   measurement covers ~11 MB and takes ~9 s on the hardware engine.
4. **Build the enclave** (`app/inference.c` + supporting code), pack it
   into the `._enclave_code` section, and HMAC-sign it with
   `tools/protect_enclave.py` as usual.
5. **Flash** with `tools/flash_n657.sh`, which writes FSBL, host bin,
   `model_bytecode.bin`, and `network_data.xSPI2.bin` to XSPI2.

The input image is generated by `tools/image_to_c_array.py` from a
COCO val2017 sample (RGB888, 224Ă—224, 150528 bytes). It is compiled
into the host as `test_image_rgb888[]` and copied into
`INPUT_SHARED + 16` at runtime.

## What the enclave does

`run_inference()` (inside `app/inference.c`):

1. Reads `INFER_MAGIC` + length from `INPUT_SHARED`.
2. Configures the NPU EPOCHCTRL controller â€” sets `ADDR` to the
   activations area, sets `CTRL.START`, drops `EC` (epoch controller)
   into running state.
3. Polls EPC.IRQ for completion and inlines the W1C ack (the IRQ never
   reaches the NVIC line in this design â€” see memory note
   `project_n657_g2b_complete.md`).
4. Copies 1470 bytes of INT8 logits (Tiny YOLO v2 raw output) from the
   NPU activations area at `OUTPUT_OFFSET = 0x67C0` into
   `OUTPUT_SHARED + 64`.
5. Writes status to `OUTPUT_SHARED[0]` and returns. SysTick may
   preempt at any point; the host re-enters until status is
   `STATUS_TERMINATED`.

The post-processing (NMS + box decode) runs **on the host**, in
`host/object_detection_n657/src/postprocess.c`. Keeping decode out of
the enclave minimizes the Secure-side TCB while still proving that the
NPU compute path is Secure.

## Performance Notes

The ~9 s wall time per inference is currently CPU-AHB MMIO bound,
not NPU-bound: the enclave polls EPC.IRQ across `200 MHz` HCLK reads,
and the NPU itself completes its 900 MHz schedule well before the
poll loop notices. Attempts to shorten the wall (SysTick bump,
IRQ-driven sync, INTCTRL masks, cache-maintenance regen, HCLK to
400 MHz) either saturated the same bottleneck or hung â€” the AHB
ceiling is 200 MHz on this part.

## Where the model came from

The `.onnx` and `_Q.json` under `Model/NUCLEO-N657X0-Q/` were produced
by the ST Edge AI toolchain (`stedgeai`) from
`quantized_tiny_yolo_v2_224__OE_3_1_0.onnx`. The toolchain is also
required to regenerate `network_ecblobs.h` / `network_data.xSPI2.bin`
with a different memory-pool layout. On macOS:

```
/Applications/ST/STEdgeAI_v2/2.0/Utilities/mac/stedgeai
```

Without the toolchain, the demo runs fine with the committed model
artifacts â€” they are tracked in this repo for reproducibility.
